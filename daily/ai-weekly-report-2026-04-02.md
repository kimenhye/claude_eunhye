## 업무 1. 다국어 지원(i18n) 리소스 생성 및 적용 (ESQ-97)

### 기존 방식
- 영문/한글 리소스 파일(en.js, ko.js)에 다국어 키를 수동으로 하나씩 추가
- XML/JS 파일 내 하드코딩된 한글 문자열을 수동으로 찾아 다국어 키로 치환
- 대상 페이지가 많을수록 누락·오타 위험 증가, 약 4~5시간 소요 예상

### AI 적용
- Claude Code로 XML/JS 파일에서 하드코딩된 한글 문자열을 자동 추출
- en.js/ko.js 리소스 키-값 쌍 일괄 생성
- mode-manager 포함 11개 파일(edgeList, notice_versionUp, popup_sender, propSetting, user_management 등)의 하드코딩 문자열을 다국어 키로 자동 치환

### 결과
- en.js 약 970줄 추가, ko.js 93줄 정비, 11개 파일 다국어 적용 완료
- 전체 작업 약 1시간 소요 (수동 시 약 4~5시간 추정, **시간 절감 약 75~80%**)
- 증적: 커밋 `0b06909`, `5ac27c0` (inswave/EdgeSquare develop 브랜치)

## 업무 2. 구버전 테이블 삭제 및 메뉴 정리 (ESQ-511)

### 기존 방식
- 6개 DB 벤더별(MariaDB, Oracle, H2, PostgreSQL, SQLServer, NHBank 전용) DDL 파일을 개별 열어 삭제 대상 테이블을 수동으로 찾아 제거
- DML 데이터도 연관 테이블 참조를 확인하며 수동 정리
- 수동 작업 시 약 1~2시간 소요 예상

### AI 적용
- Claude Code로 전체 DDL/DML 파일을 스캔하여 구버전 관련 테이블 자동 식별
- 9개 파일에서 해당 CREATE TABLE, INSERT 구문 일괄 삭제

### 결과
- 9개 DDL/DML 파일에서 약 187줄 삭제, DB 스키마 정리 완료
- 전체 작업 약 15분 소요 (수동 시 약 1~2시간 추정, **시간 절감 약 80%**)
- 증적: 커밋 `20acfd0` (inswave/EdgeSquare develop 브랜치)
