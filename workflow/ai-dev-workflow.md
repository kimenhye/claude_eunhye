# AI 기반 개발 워크플로우

> Jira 이슈를 받았을 때부터 PR 완료까지의 전체 사이클

## Phase 0. 이슈 분석 (5~10분)

Jira 이슈를 받으면 Claude Code에게 컨텍스트를 공유한다.

```
이 이슈를 분석해줘:
- Jira: [ESQ-XXX] 제목
- 요구사항: (이슈 설명 붙여넣기)
- 관련 파일: (알고 있다면)

영향 범위, 수정 대상 파일, 리스크를 정리해줘.
```

**산출물:** 영향 범위 분석 결과 (구두 확인 또는 메모)

---

## Phase 1. 설계 (15~30분)

Claude Code에게 설계 문서 작성을 요청한다.

```
[ESQ-XXX] 설계 문서를 작성해줘:
1. 현재 구조 분석 (AS-IS)
2. 변경 설계 (TO-BE)
3. 수정 대상 파일 목록
4. DB 변경사항 (DDL/DML)
5. API 변경사항 (있다면)
```

**산출물:** 설계 문서 (`docs/ESQ-XXX-design.md`)

---

## Phase 2. 설계 리뷰 (10~15분)

작성된 설계를 Claude Code에게 리뷰 요청한다.

```
이 설계 문서를 리뷰해줘. 아래 등급으로 이슈를 분류해줘:
- Critical: 구현 전 반드시 수정 필요
- Important: 수정 권장, 안 하면 기술부채
- Suggestion: 개선하면 좋지만 선택사항

특히 확인할 것:
- 기존 코드와의 호환성
- 멀티 DB 벤더 영향 (Oracle, MySQL, PostgreSQL, SQLServer, Tibero)
- 프론트/백엔드 정합성
- 누락된 엣지 케이스
```

Critical 이슈가 있으면 설계 수정 후 2차 리뷰까지 진행한다.

**산출물:** 리뷰 피드백 반영된 최종 설계

---

## Phase 3. 구현 (핵심 작업)

설계 문서를 기반으로 Claude Code에게 구현을 요청한다.

```
[ESQ-XXX] 설계 문서 기반으로 구현해줘.
- 설계: docs/ESQ-XXX-design.md
- 컴포넌트 단위로 나눠서 커밋해줘
- 커밋 메시지 형식: [ESQ-XXX][공통] 변경내용 요약
```

### 구현 원칙
- **컴포넌트 단위로 커밋** - 한 번에 몰아서 하지 않는다
- **백엔드 → SQL → 프론트엔드** 순서로 진행
- **멀티 DB 벤더 DDL은 일괄 수정** - 하나만 바꾸고 나머지 누락하지 않는다
- 구현 중 설계와 다른 부분이 생기면 설계 문서도 함께 업데이트

**산출물:** 기능 구현 코드 + 커밋 이력

---

## Phase 4. 테스트 생성 (20~30분)

구현 완료 후 Claude Code에게 테스트 작성을 요청한다.

```
[ESQ-XXX] 구현한 코드에 대해 테스트를 작성해줘:
1. 단위 테스트 - 핵심 비즈니스 로직
2. 통합 테스트 - API 엔드포인트 (있다면)
3. 엣지 케이스 - null, 빈값, 경계값, 권한 등
```

**산출물:** 테스트 코드 + 실행 결과

---

## Phase 5. 코드 리뷰 (10~15분)

전체 변경사항에 대해 Claude Code에게 코드 리뷰를 요청한다.

```
이번 [ESQ-XXX] 전체 변경사항을 코드 리뷰해줘:
- git diff develop 기준으로 확인
- Critical(C) / Important(I) / Suggestion(S) 등급으로 분류
- 보안 취약점, 성능 이슈, 코드 중복 중점 확인
```

Critical/Important 이슈는 수정 후 재리뷰한다.

**산출물:** 리뷰 이슈 수정 완료

---

## Phase 6. PR 생성 + 증적 정리 (5~10분)

```
PR을 생성해줘:
- base: develop
- 제목: [ESQ-XXX] 기능 요약
- 본문에 포함할 것:
  - 변경 요약
  - 커밋 수, 변경 파일 수, 줄 수
  - 테스트 결과
  - 설계 문서 링크
```

**산출물:** GitHub PR + 증적 링크

---

## 주간보고 연동

매주 목요일, 위 Phase를 거친 작업을 주간보고로 정리한다.

```
이번 주 작업 내역을 주간보고 양식으로 정리해줘:
- git log --since="지난주 금요일" --until="이번주 목요일" 기준
- 업무별로 기존→AI적용→결과 구조
- 커밋 수, 줄 수, 시간 절감 정량 지표 포함
- 대표 결과물 링크 포함
- agmon 실행 결과 첨부
```

|-------|------|------|--------|

| 2. 설계 리뷰 | AI 리뷰 + 피드백 반영 | 10~15분 | 리뷰 반영 설계 |
| 3. 구현 | 코드 생성 + 커밋 | 핵심 | 코드 + 커밋 |
| 4. 테스트 | 테스트 자동 생성 | 20~30분 | 테스트 코드 |
| 5. 코드 리뷰 | AI 리뷰 + 수정 | 10~15분 | 최종 코드 |
| 6. PR | PR 생성 + 증적 | 5~10분 | PR + 링크 |
