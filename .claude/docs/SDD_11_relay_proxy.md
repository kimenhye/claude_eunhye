## 1. 개요

### 1.1 목적
Edge 디바이스와 외부 시스템 간 HTTP 프록시 릴레이 및 대역폭 스로틀링 기능을 설계한다.

### 1.2 범위
- HTTP 프록시 릴레이
- 대역폭 스로틀링 (Token Bucket 알고리즘)
- 스케줄 기반 제어

---

## 2. 기능 설계

| No | 기능명 | 설명 | 우선순위 |
|----|--------|------|----------|
| 1 | HTTP 프록시 | HTTP 요청 릴레이/프록시 | 상 |
| 2 | 대역폭 제어 | Token Bucket 기반 스로틀링 | 중 |
| 3 | 스케줄 룰 | 시간대별 대역폭 스케줄 관리 | 중 |

---

## 3. 상세 설계

### 3.1 주요 클래스

| 클래스 | 설명 |
|--------|------|
| RelayProxyService | HTTP 프록시 릴레이 서비스 |
| ThrottleService | 대역폭 스로틀링 서비스 |
| SharedTokenBucket | 공유 Token Bucket 구현 |
| ThrottledOutputStream | 스로틀링 적용 출력 스트림 |
| ScheduleRule | 스케줄 기반 제어 규칙 |

### 3.2 Token Bucket 알고리즘
- 공유 Token Bucket을 통해 전체 대역폭 제한
- ThrottledOutputStream으로 스트림 레벨 스로틀링
- ScheduleRule로 시간대별 제한 조정
